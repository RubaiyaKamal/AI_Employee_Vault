# Weekly Business Audit - Gold Tier

## Title
**Weekly Business Audit** - Comprehensive Business Analysis and Reporting

## Description

This skill executes the weekly business audit system that automatically analyzes business activities, generates comprehensive reports, and identifies trends, anomalies, and improvement opportunities.

**Capability Level**: Gold Tier
**Category**: Audit and Reporting
**Risk Level**: Low (Read-only analysis)
**Estimated Duration**: 5-10 minutes

## Instructions

### Execution Flow

#### 1. Initiate Weekly Audit
```bash
# Run weekly audit for current week
node weekly_audit_runner.js

# Or run for specific week
node weekly_audit_runner.js --week-date 2026-01-06
```

**What It Does**:
- Calculates the start and end of the week (Monday to Sunday)
- Collects data from all business sources
- Analyzes the collected data
- Generates comprehensive report

#### 2. Data Collection Phase

**Data Sources**:
- **Financial Data**: Transactions, income, expenses, categories
  - Source: `./data/transactions/*.json`
  - Metrics: Total income, total expenses, net change, category breakdown

- **Communication Data**: Emails, meetings, calls
  - Source: `./data/emails/*.json`
  - Metrics: Emails sent/received, meetings attended, response times

- **Productivity Data**: Tasks, completions, focus time
  - Source: `./data/tasks/*.json`
  - Metrics: Tasks created/completed, completion rate, average time

- **Social Data**: Posts, engagement, reach
  - Source: `./data/social/*.json`
  - Metrics: Posts, likes, shares, comments, engagement rate

- **Document Data**: Created, modified, shared documents
  - Source: `./data/documents/*.json`
  - Metrics: Document counts by type and activity

#### 3. Analysis Phase

**Financial Analysis**:
- Calculate net change (income - expenses)
- Compute expense ratio
- Identify largest expense categories
- Flag if expenses exceed 80% of income

**Communication Analysis**:
- Analyze email patterns (sent vs received ratio)
- Track meeting frequency
- Calculate average response time

**Productivity Analysis**:
- Calculate task completion rate
- Identify productivity trends
- Flag low completion rates (<50%)

**Trend Identification**:
- Compare with historical data
- Identify increasing/decreasing patterns
- Calculate confidence scores

**Anomaly Detection**:
- Detect unusual spending patterns
- Flag significant deviations from averages
- Identify potential issues

### Constraints

**Time Window**: Analyzes data for the most recent complete week (Monday-Sunday)

**Data Requirements**:
- Transaction files must be in JSON format
- Email files must contain timestamp and direction fields
- Task files must contain createdAt and completedAt fields
- All dates must be in ISO 8601 format

**Performance**:
- Execution time: Target <10 minutes
- Memory usage: <500MB
- Disk space: <10MB per report

**Error Handling**:
- If data source missing: Skip that section, continue with others
- If parsing error: Log error, use default values
- If critical error: Save partial report with error information

### Scheduled Execution

**Automatic Schedule**:
- Runs every Monday at 1:00 AM
- Covers previous week (Monday-Sunday)
- Configured via cron: `0 1 * * 1`

**Manual Execution**:
- Can be run anytime via CLI
- Can specify custom week date
- Useful for historical analysis or missed audits
